3 Caracterizando tempos de execução

A ordem de crescimento do tempo de execução de um algoritmo é uma maneira simples de caracterizar sua eficiência e compará-lo com outros algoritmos. Para entradas grandes o suficiente, as constantes multiplicativas e os termos de ordem inferior de um tempo de execução exato são dominados pelos efeitos do tamanho da entrada em si.

A eficiência assintótica está preocupada com o aumento do tempo de execução de um algoritmo à medida que o tamanho da entrada aumenta indefinidamente. Um algoritmo que é assintoticamente mais eficiente geralmente é a melhor escolha para todas as entradas, exceto as muito pequenas.

3.1 Notações assintóticas $\Omega$, $O$ e $\Theta$

As notações assintóticas são projetadas para caracterizar funções em geral, não apenas os tempos de execução dos algoritmos. Elas podem ser usadas para descrever outros aspectos dos algoritmos, como a quantidade de espaço que eles usam.

A notação $O$ caracteriza um limite superior no comportamento assintótico de uma função, baseado no termo de maior ordem. Ela diz que uma função cresce no máximo a uma certa taxa (um algoritmo não vai gastar mais que essa quantidade de instruções). A função $7n^3+100n^2-20n+6$ é $O(n^3)$ assim como é $O(n^4)$. Ou seja, é $O(n^c)$ para qualquer constante $c\ge 3$.

A notação $\Omega$ caracteriza um limite inferior no comportamento assintótico de uma função, baseado no termo de maior ordem. Ela diz que uma função cresce pelo menos tão rápido quanto uma certa taxa (um algoritmo não vai gastar menos que essa quantidade de instruções). A mesma função é $\Omega(n^3)$ assim como é $\Omega(n^2)$. Ou seja, é $\Omega(n^c)$ para $c\le 3$.

A notação $\Theta$ caracteriza um limite apertado no comportamento assintótico de uma função, baseado no termo de maior ordem. Ela diz que uma função cresce precisamente a uma certa taxa, abaixo de um fator constante e acima de outro fator constante.

Se você puder mostrar que uma função é tanto $O(f(n))$ quanto $\Omega(f(n))$ para alguma função $f(n)$, então você mostrou que a função é $\Theta(f(n))$.

Revisitando o insertion sort

Considerando o cenário do pior caso. O tempo de execução é dominado pelo loop interno. Como cada uma das $n - 1$ iterações do loop externo faz com que o loop interno itere no máximo $i - 1$ vezes, e como $i$ é no máximo $n$, o número total de iterações do loop interno é no máximo $(n - 1)(n - 1)$, o que é menor que $n^2$. Como cada iteração do loop interno leva tempo constante, o tempo total gasto no loop interno é no máximo uma constante vezes $n^2$, ou $O(n^2)$.

O tempo de execução do pior caso do algoritmo insertion sort é $\Omega(n^2)$. Isso significa que para cada tamanho de entrada $n$ acima de um certo limiar, existe pelo menos uma entrada de tamanho $n$ para a qual o algoritmo leva pelo menos $cn^2$ tempo, para alguma constante positiva $c$.

Vamos supor que $n$ seja múltiplo de $3$ e dividir o array $A$ em três partes. Se os $n/3$ maiores valores ocuparem as $n/3$ primeiras posições do array, eles devem passar por cada uma das $n/3$ posições intermediárias para acabar nas últimas $n/3$ posições. Cada valor passa por essas posições intermediárias uma posição por vez, por pelo menos $n/3$ execuções da linha 6. Como pelo menos $n/3$ valores têm que passar por pelo menos $n/3$ posições, o tempo gasto pelo insertion sort no pior caso é pelo menos proporcional a $(n/3)(n/3) = n^2/9$, que é $\Omega(n^2)$.

3.2 Notação assintótica: definições formais

Notação $O$

Para uma função $g(n)$ dada, denotamos por $O(g(n))$ o conjunto de funções

$O(g(n))=\lbrace f(n) : \text{existem constantes positivas } c \text{ e } n_0 \text{ tal que } 0 \le f(n) \le cg(n) \text{ para todo } n \ge n_0 \rbrace$.

A Figura 3.2(a) mostra a intuição por trás da notação $O$. Para todos os valores de $n$ em e à direita de $n_0$, o valor da função $f(n)$ está na linha ou abaixo de $cg(n)$.

A definição de $O(g(n))$, assim como as outras notações, requer que cada função $f(n)$ do conjunto $O(g(n))$ seja assintoticamente não negativa: $f(n)$ deve ser não negativa sempre que $n$ for suficientemente grande.

Por que podemos ignorar os termos de menor ordem e o coeficiente constante do termo de maior ordem? Por exemplo, $4n^2 + 100n + 500 = O(n^2)$, mesmo que os termos de menor ordem tenham coeficientes muito maiores. Precisamos encontrar constantes positivas $c$ e $n_0$ para as quais $4n^2 + 100n + 500 \le cn^2$ para todo $n \ge n_0$. Dividindo ambos os lados por $n^2$ temos $4 + 100/n + 500/n^2 \le c$. Essa inequação é válida para muitas escolhas de $c$ e $n_0$, como $n_0 = 10$, $c = 19$, ou $n_0 = 100$, $c = 5.05$.

Notação $\Omega$

Para uma função $g(n)$ dada, denotamos por $\Omega(g(n))$ o conjunto de funções

$\Omega(g(n))=\lbrace f(n) : \text{existem constantes positivas } c \text{ e } n_0 \text{ tal que } 0 \le cg(n) \le f(n) \text{ para todo } n \ge n_0 \rbrace$.

A Figura 3.2(b) mostra a intuição por trás da notação $\Omega$. Para todos os valores de $n$ em e à direita de $n_0$, o valor da função $f(n)$ está acima ou igual a $cg(n)$.

Da mesma forma, $4n^2 + 100n + 500 = \Omega(n^2)$. Essa inequação vale quando $n_0$ é um número inteiro qualquer e $c = 4$.

Notação $\Theta$

Para uma função $g(n)$ dada, denotamos por $\Theta(g(n))$ o conjunto de funções

$\Theta(g(n))=\lbrace f(n) : \text{existem constantes positivas } c_1 \text{, } c_2 \text{ e } n_0 \text{ tal que } 0 \le c_1g(n) \le f(n) \le c_2g(n) \text{ para todo } n \ge n_0 \rbrace$.

A Figura 3.2(c) mostra a intuição por trás da notação $\Theta$. Para todos os valores de $n$ em e à direita de $n_0$, o valor de $f(n)$ está acima ou igual a $c_1g(n)$ e abaixo ou igual a $c_2g(n)$.

Podemos concluir:

Teorema 3.1: Para quaisquer duas funções $f(n)$ e $g(n)$, temos que $f(n) = \Theta(g(n))$ se, e somente se, $f(n) = O(g(n))$ e $f(n) = \Omega(g(n))$.

Notação assintótica e tempos de execução

Quando caracterizamos o tempo de execução de um algoritmo usando notação assintótica, é importante ser o mais preciso possível sem exagerar no tempo de execução. Por exemplo, o tempo de execução do pior caso do algoritmo insertion sort pode ser corretamente declarado como $O(n^2)$, $\Omega(n^2)$ e $\Theta(n^2)$, sendo $\Theta(n^2)$ o mais preciso e preferido. O tempo de execução no melhor caso pode ser corretamente declarado como $O(n)$, $\Omega(n)$ e $\Theta(n)$, sendo $\Theta(n)$ o mais preciso e recomendado.

No entanto, seria incorreto dizer que o tempo de execução do algoritmo insertion sort é $\Theta(n^2)$ sem especificar "pior caso", pois isso implica que ele é executado em tempo $\Theta(n^2)$ em todos os casos. Seria correto dizer que o tempo de execução é $O(n^2)$, pois o tempo de execução cresce não mais rápido que $n^2$ em todos os casos. Da mesma forma, seria incorreto dizer que o tempo de execução é $\Theta(n)$, mas seria correto dizer que o tempo de execução é $\Omega(n)$.

Agora, como o algoritmo merge sort executa em tempo $\Theta(n\log n)$ em todos os casos, podemos simplesmente dizer que seu tempo de execução é $\Theta(n\log n)$ sem especificar o pior caso, o melhor caso ou qualquer outro caso.

Se um algoritmo tem um tempo de execução de $3n^2 + 20n$ em todos os casos, podemos usar notação assintótica para escrever que seu tempo de execução é $O(n^3)$, embora seja melhor dizer $\Theta(n^2)$.

Notação assintótica em equações e inequações

A notação assintótica é formalmente definida em termos de conjuntos, mas muitas vezes é usada com o sinal de igual (=) em vez do sinal de pertence ($\in$) dentro das fórmulas. Quando a notação assintótica aparece sozinha no lado direito de uma equação ou inequação, o sinal de igual significa pertence ao conjunto. Assim, $4n^2 + 100n + 500 = O(n^2)$ é equivalente a $4n^2 + 100n + 500 \in O(n^2)$

No entanto, quando a notação assintótica aparece dentro de uma fórmula, ela é interpretada como uma função anônima, que não nos preocupamos em nomeá-la, como $2n^2 + 3n + 1 = 2n^2 + \Theta(n)$. Isso pode ajudar a eliminar detalhes desnecessários em uma equação.

O número de funções anônimas em uma expressão é igual ao número de vezes que a notação assintótica aparece. Portanto, a expressão $\sum_{i=0}^n O(i)$ possui apenas uma única função anônima. Atenção! Não é a mesma coisa que $O(1)+O(2)+\dots+O(n)$, mas é igual a $O(n^2)$.

Quando a notação assintótica aparece no lado esquerdo e direito de uma equação, significa que, não importa como as funções anônimas são escolhidas no lado esquerdo, há uma maneira de escolher as funções anônimas no lado direito para tornar a equação válida. Por exemplo, dado $2n^2+\Theta(n)=\Theta(n^2)$, para qualquer função $f(n) \in \Theta(n)$, há uma função $g(n) \in \Theta(n^2)$ tal que $2n^2+f(n)=g(n)$ para todos os $n$.

Abusos adequados da notação assintótica

Além do abuso da igualdade para significar pertencimento ao conjunto, outro abuso da notação assintótica ocorre quando a variável tendendo a ∞ deve ser inferida do contexto, como quando a função dentro da notação assintótica é uma constante. Por exemplo, se $f(n) = O(1)$, fica claro que a variável de interesse é $n$.

Quando a função dentro da notação assintótica é limitada por uma constante positiva, a notação assintótica pode ser usada de outra forma, especialmente ao declarar recorrências. Por exemplo, a declaração $T(n) = O(1)$ para $n \lt 3$ é tecnicamente sem sentido de acordo com a definição formal da notação $O$, mas convencionalmente significa que existe uma constante positiva $c$ tal que $T(n) \leq c$ para $n \lt 3$. Essa convenção nos poupa do trabalho de nomear a constante limitante.

Ocasionalmente, a função que descreve o tempo de execução de um algoritmo pode não estar definida para determinados tamanhos de entrada. Por exemplo, quando um algoritmo assume que o tamanho da entrada é uma potência exata de 2. Nesses casos, ainda usamos a notação assintótica para descrever o crescimento do tempo de execução, entendendo que quaisquer restrições se aplicam apenas quando a função estiver definida.

Na matemática, é admissível abusar de uma notação desde que não a usemos de forma inadequada e entendamos precisamente seu significado a esse uso ampliado.

Notações $o$ e $\omega$

$o(g(n)) = \lbrace f(n) : \text{ para qualquer constante positiva } c \gt 0 \text{, existe uma constante } n_0 \gt 0 \text{ tal que } 0 \le f(n) \lt cg(n) \text{ para todo } n \ge n_0 \rbrace$.

$\omega(g(n)) = \lbrace f(n) : \text{ para qualquer constante positiva } c \gt 0 \text{, existe uma constante } n_0 \gt 0 \text{ tal que } 0 \le cg(n) \lt f(n) \text{ para todo } n \ge n_0 \rbrace$.

Comparando funções

Muitas das propriedades relacionais dos números reais também se aplicam às comparações assintóticas.

Transitividade:

  • $f(n) = \Theta(g(n))$ e $g(n) = \Theta(h(n))$ implicam $f(n) = \Theta(h(n))$.
  • $f(n) = O(g(n))$ e $g(n) = O(h(n))$ implicam $f(n) = O(h(n))$.
  • $f(n) = \Omega(g(n))$ e $g(n) = \Omega(h(n))$ implicam $f(n) = \Omega(h(n))$.
  • $f(n) = o(g(n))$ e $g(n) = o(h(n))$ implicam $f(n) = o(h(n))$.
  • $f(n) = \omega(g(n))$ e $g(n) = \omega(h(n))$ implicam $f(n) = \omega(h(n))$.

Reflexividade:

  • $f(n) = \Theta(f(n))$.
  • $f(n) = O(f(n))$.
  • $f(n) = \Omega(f(n))$.

Simetria:

  • $f(n) = \Theta(g(n))$ se e somente se $g(n) = \Theta(f (n))$.

Simetria transposta:

  • $f(n) = O(g (n))$ se e somente se $g (n) = \Omega(f (n))$.

Podemos fazer uma analogia entre a comparação assintótica de duas funções $f$ e $g$ e dois números reais $a$ e $b$:

  • $f (n) = O(g(n))$ é como $a \le b$.
  • $f (n) = \Omega(g(n))$ é como $a \ge b$.
  • $f (n) = \Theta(g(n))$ é como $a = b$.
  • $f (n) = o(g(n))$ é como $a \lt b$.
  • $f (n) = \omega(g(n))$ é como $a \gt b$.

3.3 Notações padrão e funções comuns

Monotonicidade

Uma função é estritamente crescente se, para todos os valores de $x$ e $y$ tal que $y > x$, temos $f(y) > f(x)$. Isso significa que, à medida que o valor de entrada aumenta, o valor da função também aumenta. Por outro lado, uma função é monótona crescente se, para todos os valores de $x$ e $y$ tal que $y > x$, temos $f(y) \geq f(x)$. Isso significa que, à medida que o valor de entrada aumenta, o valor da função não diminui.

Pisos e tetos

Para qualquer inteiro $n$, temos

$$\lfloor n \rfloor = n = \lceil n \rceil \tag{3.1}$$

Para todos os reais $x$, temos

$$x-1 \lt \lfloor x \rfloor \le x \le \lceil x \rceil \lt x+1 \tag{3.2}$$

$$-\lfloor x \rfloor = \lceil -x \rceil \tag{3.3}$$

$$-\lceil x \rceil = \lfloor -x \rfloor \tag{3.4}$$

Para qualquer número real $x\ge 0$ e inteiros $a,b\gt 0$, temos

$$\lceil \dfrac{\lceil x/a \rceil}{b}\rceil = \lceil \dfrac{x}{ab}\rceil \tag{3.5}$$

$$\lfloor \dfrac{\lfloor x/a \rfloor}{b}\rfloor = \lfloor \dfrac{x}{ab}\rfloor \tag{3.6}$$

$$\lceil \dfrac{a}{b} \rceil \le \dfrac{a+(b-1)}{b} \tag{3.7}$$

$$\lfloor \dfrac{a}{b} \rfloor \ge \dfrac{a-(b-1)}{b} \tag{3.7}$$

Para qualquer inteiro $n$ e número real $x$, temos

$$\lfloor n+x \rfloor = n + \lfloor x \rfloor \tag{3.8}$$

$$\lceil n+x \rceil = n + \lceil x \rceil \tag{3.9}$$

Aritmética modular

Para qualquer inteiro $a$ e qualquer inteiro positivo $n$, o valor de $a\bmod n$ é o resto do quociente $a/n$:

$$a\bmod n = a - n\lfloor a / n\rfloor\tag{3.11}$$

$$0\le a\bmod n \lt n\tag{3.12}$$

Se $(a\bmod n) = (b\bmod n)$, escrevemos $a = b\pmod n$, ou seja, $a$ e $b$ têm o mesmo resto quando divididos por $n$. Equivalentemente, $n$ é um divisor de $b-a$.

Polinomiais

Dizemos que uma função $f(n)$ é limitada por um polinômio se $f(n) = O(n^k)$ para alguma constante $k$.

Exponenciais

Para todos os reais $a > 0$, $m$ e $n$, temos as seguintes identidades:

  • $a^0=1$
  • $a^1=a$
  • $a^{-1}=1/a$
  • $(a^m)^n=a^{mn}$
  • $(a^m)^n=(a^n)^m$
  • $a^ma^n=a^{m+n}$
  • $a^m/a^n=a^{m-n}$
  • $(ab)^n=a^nb^n$
  • $(a/b)^n=a^n/b^n$
  • $(a/b)^{-n}=(b/a)^n$
  • $a^{m/n}=\sqrt[n]{a^m}$

Para todas as constantes reais $a > 1$ e $b$, temos:

$$n^b=o(a^n)\tag{3.13}$$

Seja $e$ o número de Euler. Para todos os reais $x$, temos:

$$e^x=1+x+\dfrac{x^2}{2!}+\dfrac{x^3}{3!}+\dots=\sum_{i=0}^\infin\dfrac{x^i}{i!}$$

$$1+x\le e^x\tag{3.14}$$

$$1+x\le e^x\le 1+x+x^2\tag{3.15}$$

$$\lim_{n\to\infin}(1+\dfrac{x}{n})^n=e^x\tag{3.16}$$

Logaritmos

Para todos os reais $a > 0$, $b > 0$, $c > 0$ e $n$, temos:

$$a=b^{\log_ba}\tag{3.17}$$

$$\log_c(ab)=\log_ca+\log_cb\tag{3.18}$$

$$\log_ba^n=n\log_ba$$

$$\log_ba=\dfrac{\log_ca}{\log_cb}\tag{3.19}$$

$$\log_b(1/a)=-\log_ba\tag{3.20}$$

$$\log_ba=\dfrac{1}{\log_ab}$$

$$a^{\log_bc}=c^{\log_ba}\tag{3.21}$$

Quando $|x|\lt 1$, temos:

$$\ln(1+x)=x-\dfrac{x^2}{2}+\dfrac{x^3}{3}-\dfrac{x^4}{4}+\dfrac{x^5}{5}-\dots \tag{3.22}$$

Para $x>-1$, temos:

$$\dfrac{x}{1+x}\le \ln(1+x)\le x \tag{3.23}$$

Dizemos que uma função $f(n)$ é limitada por um polilogaritmo se $f(n) = O(\lg^k n)$ para alguma constante $k$.

Para todas as constantes reais $a > 0$ e $b$, temos:

$$\lg^bn=(\lg n)^b=o(n^a)\tag{3.24}$$

Dessa forma, qualquer função polinomial positiva cresce mais rápido do que qualquer função polilogarítmica.

Fatoriais

A aproximação de Stirling é dada pela fórmula

$$n! = \sqrt{2\pi n} \left(\frac{n}{e}\right)^n \left(1+\Theta\left(\frac{1}{n}\right)\right) \tag{3.25}$$

$$n!=o(n^n) \tag{3.26}$$

$$n!=\omega(2^n) \tag{3.27}$$

$$\lg(n!)=\Theta(n\lg n) \tag{3.28}$$

Para todos $n\ge1$, temos

$$n! = \sqrt{2\pi n} \left(\frac{n}{e}\right)^n e^{\alpha_n} \tag{3.29}$$

onde

$$\dfrac{1}{12n+1}\lt\alpha_n\lt\dfrac{1}{12n}$$

Iteração funcional

Usamos a notação $f^{(i)}(n)$ para denotar a função $f(n)$ aplicada iterativamente $i$ vezes a um valor inicial de $n$, ou seja,

$$ \begin{aligned} f^{(i)}(n) &= \begin{cases} n & \text{se } i = 0 \\ f(f^{(i-1)}(n)) & \text{se } i \gt 0 \end{cases} \\ \end{aligned} $$

Por exemplo, se $f(n) = 2n$, então $f^{(i)}(n) = 2^in$.

Números de Fibonacci

$$ \begin{aligned} F_i &= \begin{cases} 0 & \text{se } i = 0 \\ 1 & \text{se } i = 1 \\ F_{i-1}+F_{i-2} & \text{se } i \ge 2 \end{cases} \\ \end{aligned} $$

Números de Fibonacci estão relacionados à proporção áurea $\phi$ e seu conjugado $\widehat{\phi}$, que são as duas raízes da equação $x^2 = x + 1$.

$$\phi=\dfrac{1+\sqrt 5}{2}=1.61803\cdots \tag{3.32}$$

$$\widehat{\phi}=\dfrac{1-\sqrt 5}{2}=-.61803\cdots \tag{3.33}$$

$$F_i= \dfrac{\phi^i-\widehat{\phi}^i}{\sqrt 5} = \left\lfloor \dfrac{\phi^i}{\sqrt 5}+\dfrac 1 2 \right\rfloor \tag{3.34}$$

Ou seja, o $i$-ésimo número de Fibonacci $F_i$ é igual a $\phi/\sqrt 5$ arredondado para o inteiro mais próximo. Assim, os números de Fibonacci crescem exponencialmente.

    A última inequação segue do fato que $(k/(k+1))^k\lt1$, então para todo $x$ positivo, $x(k/(k+1))^k \lt x$.

    Conclusão: Completando o passo indutivo, mostramos que se a inequação vale para $k$, ela também vale para $k + 1$. Visto que verificamos o caso base e estabelecemos o passo indutivo, pelo princípio da indução matemática, podemos concluir que $n!=o(n^n)$.

  9. Mostre que $k\lg k = \Theta(n)$ implica $k = \Theta(n / \lg n)$.

    10. Pela simetria de $\Theta$,

    $$k\lg k = \Theta(n) \Rightarrow n = \Theta(k\lg k)$$

    Uma vez que $n = \Theta(k\lg k)$, existem constantes positivas $c_1$, $c_2$ e $n_0$ tais que, para todo $n \geq n_0$, temos

    $$ \begin{gathered} c_1k\lg k \le n \le c_2 k \lg k \\ \lg(c_1k\lg k) \le \lg n \le \lg(c_2 k \lg k) && \text{(aplicado lg)} \\ \lg c_1 + \lg k + \lg \lg k \le \lg n \le \lg c_2 + \lg k + \lg \lg k && \text{(aplicada a equação (3.18))} \\ c_3 \lg k \le \lg n \le c_4 \lg k && \text{(desconsiderados menores termos)}\\ \lg n = \Theta(\lg k) \end{gathered} $$

    Uma vez que $k \lg k = \Theta(n)$, existem constantes positivas $c_1$, $c_2$ e $n_0$ tais que, para todo $n \geq n_0$, temos

    $$ \begin{aligned} c_1n &\leq k\lg k &\leq c_2n \\ \dfrac{c_1n}{\lg k} &\leq k &\leq \dfrac{c_2n}{\lg k} && \text{(dividido por $\lg k$)} \end{aligned} $$

    Uma vez que $\lg n = \Theta(\lg k)$, existem constantes positivas $d_1$, $d_2$ e $n_1$ tais que, para todo $n \geq n_1$, temos

    $d_1\lg k \leq \lg n \leq d_2\lg k$

    Isolando $\lg k$, temos $\lg k \le \dfrac{\lg n}{d_1}$ e $\lg k \ge \dfrac{\lg n}{d_2}$

    Uma vez que um denominador maior resulta em uma fração menor, e vice-versa, temos

    $\dfrac{c_1n}{\lg n / d_2} \le \dfrac{c_1n}{\lg k} \leq k \leq \dfrac{c_2n}{\lg k} \le \dfrac{c_2n}{\lg n / d_1}$

    Substituindo essas limitações na inequação anterior, obtemos

    $\dfrac{c_1d_2n}{\lg n} \leq k \leq \dfrac{c_2d_1n}{\lg n}$

    O que implica que $k = \Theta(n / \lg n)$.
