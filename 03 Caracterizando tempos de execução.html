3 Caracterizando tempos de execuo

A ordem de crescimento do tempo de execuo de um algoritmo uma maneira simples de caracterizar sua eficincia e compar-lo com outros algoritmos. Para entradas grandes o suficiente, as constantes multiplicativas e os termos de ordem inferior de um tempo de execuo exato so dominados pelos efeitos do tamanho da entrada em si.

A eficincia assinttica est preocupada com o aumento do tempo de execuo de um algoritmo medida que o tamanho da entrada aumenta indefinidamente. Um algoritmo que assintoticamente mais eficiente geralmente a melhor escolha para todas as entradas, exceto as muito pequenas.

3.1 Notaes assintticas $\Omega$, $O$ e $\Theta$

As notaes assintticas so projetadas para caracterizar funes em geral, no apenas os tempos de execuo dos algoritmos. Elas podem ser usadas para descrever outros aspectos dos algoritmos, como a quantidade de espao que eles usam.

A notao $O$ caracteriza um limite superior no comportamento assinttico de uma funo, baseado no termo de maior ordem. Ela diz que uma funo cresce no mximo a uma certa taxa (um algoritmo no vai gastar mais que essa quantidade de instrues). A funo $7n^3+100n^2-20n+6$ $O(n^3)$ assim como $O(n^4)$. Ou seja, $O(n^c)$ para qualquer constante $c\ge 3$.

A notao $\Omega$ caracteriza um limite inferior no comportamento assinttico de uma funo, baseado no termo de maior ordem. Ela diz que uma funo cresce pelo menos to rpido quanto uma certa taxa (um algoritmo no vai gastar menos que essa quantidade de instrues). A mesma funo $\Omega(n^3)$ assim como $\Omega(n^2)$. Ou seja, $\Omega(n^c)$ para $c\le 3$.

A notao $\Theta$ caracteriza um limite apertado no comportamento assinttico de uma funo, baseado no termo de maior ordem. Ela diz que uma funo cresce precisamente a uma certa taxa, abaixo de um fator constante e acima de outro fator constante.

Se voc puder mostrar que uma funo tanto $O(f(n))$ quanto $\Omega(f(n))$ para alguma funo $f(n)$, ento voc mostrou que a funo $\Theta(f(n))$.

Revisitando o insertion sort

Considerando o cenrio do pior caso. O tempo de execuo dominado pelo loop interno. Como cada uma das $n - 1$ iteraes do loop externo faz com que o loop interno itere no mximo $i - 1$ vezes, e como $i$ no mximo $n$, o nmero total de iteraes do loop interno no mximo $(n - 1)(n - 1)$, o que menor que $n^2$. Como cada iterao do loop interno leva tempo constante, o tempo total gasto no loop interno no mximo uma constante vezes $n^2$, ou $O(n^2)$.

O tempo de execuo do pior caso do algoritmo insertion sort $\Omega(n^2)$. Isso significa que para cada tamanho de entrada $n$ acima de um certo limiar, existe pelo menos uma entrada de tamanho $n$ para a qual o algoritmo leva pelo menos $cn^2$ tempo, para alguma constante positiva $c$.

Vamos supor que $n$ seja mltiplo de $3$ e dividir o array $A$ em trs partes. Se os $n/3$ maiores valores ocuparem as $n/3$ primeiras posies do array, eles devem passar por cada uma das $n/3$ posies intermedirias para acabar nas ltimas $n/3$ posies. Cada valor passa por essas posies intermedirias uma posio por vez, por pelo menos $n/3$ execues da linha 6. Como pelo menos $n/3$ valores tm que passar por pelo menos $n/3$ posies, o tempo gasto pelo insertion sort no pior caso pelo menos proporcional a $(n/3)(n/3) = n^2/9$, que $\Omega(n^2)$.

3.2 Notao assinttica: definies formais

Notao $O$

Para uma funo $g(n)$ dada, denotamos por $O(g(n))$ o conjunto de funes

$O(g(n))=\lbrace f(n) : \text{existem constantes positivas } c \text{ e } n_0 \text{ tal que } 0 \le f(n) \le cg(n) \text{ para todo } n \ge n_0 \rbrace$.

A Figura 3.2(a) mostra a intuio por trs da notao $O$. Para todos os valores de $n$ em e direita de $n_0$, o valor da funo $f(n)$ est na linha ou abaixo de $cg(n)$.

A definio de $O(g(n))$, assim como as outras notaes, requer que cada funo $f(n)$ do conjunto $O(g(n))$ seja assintoticamente no negativa: $f(n)$ deve ser no negativa sempre que $n$ for suficientemente grande.

Por que podemos ignorar os termos de menor ordem e o coeficiente constante do termo de maior ordem? Por exemplo, $4n^2 + 100n + 500 = O(n^2)$, mesmo que os termos de menor ordem tenham coeficientes muito maiores. Precisamos encontrar constantes positivas $c$ e $n_0$ para as quais $4n^2 + 100n + 500 \le cn^2$ para todo $n \ge n_0$. Dividindo ambos os lados por $n^2$ temos $4 + 100/n + 500/n^2 \le c$. Essa inequao vlida para muitas escolhas de $c$ e $n_0$, como $n_0 = 10$, $c = 19$, ou $n_0 = 100$, $c = 5.05$.

Notao $\Omega$

Para uma funo $g(n)$ dada, denotamos por $\Omega(g(n))$ o conjunto de funes

$\Omega(g(n))=\lbrace f(n) : \text{existem constantes positivas } c \text{ e } n_0 \text{ tal que } 0 \le cg(n) \le f(n) \text{ para todo } n \ge n_0 \rbrace$.

A Figura 3.2(b) mostra a intuio por trs da notao $\Omega$. Para todos os valores de $n$ em e direita de $n_0$, o valor da funo $f(n)$ est acima ou igual a $cg(n)$.

Da mesma forma, $4n^2 + 100n + 500 = \Omega(n^2)$. Essa inequao vale quando $n_0$ um nmero inteiro qualquer e $c = 4$.

Notao $\Theta$

Para uma funo $g(n)$ dada, denotamos por $\Theta(g(n))$ o conjunto de funes

$\Theta(g(n))=\lbrace f(n) : \text{existem constantes positivas } c_1 \text{, } c_2 \text{ e } n_0 \text{ tal que } 0 \le c_1g(n) \le f(n) \le c_2g(n) \text{ para todo } n \ge n_0 \rbrace$.

A Figura 3.2(c) mostra a intuio por trs da notao $\Theta$. Para todos os valores de $n$ em e direita de $n_0$, o valor de $f(n)$ est acima ou igual a $c_1g(n)$ e abaixo ou igual a $c_2g(n)$.

Podemos concluir:

Teorema 3.1: Para quaisquer duas funes $f(n)$ e $g(n)$, temos que $f(n) = \Theta(g(n))$ se, e somente se, $f(n) = O(g(n))$ e $f(n) = \Omega(g(n))$.

Notao assinttica e tempos de execuo

Quando caracterizamos o tempo de execuo de um algoritmo usando notao assinttica, importante ser o mais preciso possvel sem exagerar no tempo de execuo. Por exemplo, o tempo de execuo do pior caso do algoritmo insertion sort pode ser corretamente declarado como $O(n^2)$, $\Omega(n^2)$ e $\Theta(n^2)$, sendo $\Theta(n^2)$ o mais preciso e preferido. O tempo de execuo no melhor caso pode ser corretamente declarado como $O(n)$, $\Omega(n)$ e $\Theta(n)$, sendo $\Theta(n)$ o mais preciso e recomendado.

No entanto, seria incorreto dizer que o tempo de execuo do algoritmo insertion sort $\Theta(n^2)$ sem especificar "pior caso", pois isso implica que ele executado em tempo $\Theta(n^2)$ em todos os casos. Seria correto dizer que o tempo de execuo $O(n^2)$, pois o tempo de execuo cresce no mais rpido que $n^2$ em todos os casos. Da mesma forma, seria incorreto dizer que o tempo de execuo $\Theta(n)$, mas seria correto dizer que o tempo de execuo $\Omega(n)$.

Agora, como o algoritmo merge sort executa em tempo $\Theta(n\log n)$ em todos os casos, podemos simplesmente dizer que seu tempo de execuo $\Theta(n\log n)$ sem especificar o pior caso, o melhor caso ou qualquer outro caso.

Se um algoritmo tem um tempo de execuo de $3n^2 + 20n$ em todos os casos, podemos usar notao assinttica para escrever que seu tempo de execuo $O(n^3)$, embora seja melhor dizer $\Theta(n^2)$.

Notao assinttica em equaes e inequaes

A notao assinttica formalmente definida em termos de conjuntos, mas muitas vezes usada com o sinal de igual (=) em vez do sinal de pertence ($\in$) dentro das frmulas. Quando a notao assinttica aparece sozinha no lado direito de uma equao ou inequao, o sinal de igual significa pertence ao conjunto. Assim, $4n^2 + 100n + 500 = O(n^2)$ equivalente a $4n^2 + 100n + 500 \in O(n^2)$

No entanto, quando a notao assinttica aparece dentro de uma frmula, ela interpretada como uma funo annima, que no nos preocupamos em nome-la, como $2n^2 + 3n + 1 = 2n^2 + \Theta(n)$. Isso pode ajudar a eliminar detalhes desnecessrios em uma equao.

O nmero de funes annimas em uma expresso igual ao nmero de vezes que a notao assinttica aparece. Portanto, a expresso $\sum_{i=0}^n O(i)$ possui apenas uma nica funo annima. Ateno! No a mesma coisa que $O(1)+O(2)+\dots+O(n)$, mas igual a $O(n^2)$.

Quando a notao assinttica aparece no lado esquerdo e direito de uma equao, significa que, no importa como as funes annimas so escolhidas no lado esquerdo, h uma maneira de escolher as funes annimas no lado direito para tornar a equao vlida. Por exemplo, dado $2n^2+\Theta(n)=\Theta(n^2)$, para qualquer funo $f(n) \in \Theta(n)$, h uma funo $g(n) \in \Theta(n^2)$ tal que $2n^2+f(n)=g(n)$ para todos os $n$.

Abusos adequados da notao assinttica

Alm do abuso da igualdade para significar pertencimento ao conjunto, outro abuso da notao assinttica ocorre quando a varivel tendendo a deve ser inferida do contexto, como quando a funo dentro da notao assinttica uma constante. Por exemplo, se $f(n) = O(1)$, fica claro que a varivel de interesse $n$.

Quando a funo dentro da notao assinttica limitada por uma constante positiva, a notao assinttica pode ser usada de outra forma, especialmente ao declarar recorrncias. Por exemplo, a declarao $T(n) = O(1)$ para $n \lt 3$ tecnicamente sem sentido de acordo com a definio formal da notao $O$, mas convencionalmente significa que existe uma constante positiva $c$ tal que $T(n) \leq c$ para $n \lt 3$. Essa conveno nos poupa do trabalho de nomear a constante limitante.

Ocasionalmente, a funo que descreve o tempo de execuo de um algoritmo pode no estar definida para determinados tamanhos de entrada. Por exemplo, quando um algoritmo assume que o tamanho da entrada uma potncia exata de 2. Nesses casos, ainda usamos a notao assinttica para descrever o crescimento do tempo de execuo, entendendo que quaisquer restries se aplicam apenas quando a funo estiver definida.

Na matemtica, admissvel abusar de uma notao desde que no a usemos de forma inadequada e entendamos precisamente seu significado a esse uso ampliado.

Notaes $o$ e $\omega$

$o(g(n)) = \lbrace f(n) : \text{ para qualquer constante positiva } c \gt 0 \text{, existe uma constante } n_0 \gt 0 \text{ tal que } 0 \le f(n) \lt cg(n) \text{ para todo } n \ge n_0 \rbrace$.

$\omega(g(n)) = \lbrace f(n) : \text{ para qualquer constante positiva } c \gt 0 \text{, existe uma constante } n_0 \gt 0 \text{ tal que } 0 \le cg(n) \lt f(n) \text{ para todo } n \ge n_0 \rbrace$.

Comparando funes

Muitas das propriedades relacionais dos nmeros reais tambm se aplicam s comparaes assintticas.

Transitividade:

  • $f(n) = \Theta(g(n))$ e $g(n) = \Theta(h(n))$ implicam $f(n) = \Theta(h(n))$.
  • $f(n) = O(g(n))$ e $g(n) = O(h(n))$ implicam $f(n) = O(h(n))$.
  • $f(n) = \Omega(g(n))$ e $g(n) = \Omega(h(n))$ implicam $f(n) = \Omega(h(n))$.
  • $f(n) = o(g(n))$ e $g(n) = o(h(n))$ implicam $f(n) = o(h(n))$.
  • $f(n) = \omega(g(n))$ e $g(n) = \omega(h(n))$ implicam $f(n) = \omega(h(n))$.

Reflexividade:

  • $f(n) = \Theta(f(n))$.
  • $f(n) = O(f(n))$.
  • $f(n) = \Omega(f(n))$.

Simetria:

  • $f(n) = \Theta(g(n))$ se e somente se $g(n) = \Theta(f (n))$.

Simetria transposta:

  • $f(n) = O(g (n))$ se e somente se $g (n) = \Omega(f (n))$.

Podemos fazer uma analogia entre a comparao assinttica de duas funes $f$ e $g$ e dois nmeros reais $a$ e $b$:

  • $f (n) = O(g(n))$ como $a \le b$.
  • $f (n) = \Omega(g(n))$ como $a \ge b$.
  • $f (n) = \Theta(g(n))$ como $a = b$.
  • $f (n) = o(g(n))$ como $a \lt b$.
  • $f (n) = \omega(g(n))$ como $a \gt b$.

3.3 Notaes padro e funes comuns

Monotonicidade

Uma funo estritamente crescente se, para todos os valores de $x$ e $y$ tal que $y > x$, temos $f(y) > f(x)$. Isso significa que, medida que o valor de entrada aumenta, o valor da funo tambm aumenta. Por outro lado, uma funo montona crescente se, para todos os valores de $x$ e $y$ tal que $y > x$, temos $f(y) \geq f(x)$. Isso significa que, medida que o valor de entrada aumenta, o valor da funo no diminui.

Pisos e tetos

Para qualquer inteiro $n$, temos

$$\lfloor n \rfloor = n = \lceil n \rceil \tag{3.1}$$

Para todos os reais $x$, temos

$$x-1 \lt \lfloor x \rfloor \le x \le \lceil x \rceil \lt x+1 \tag{3.2}$$

$$-\lfloor x \rfloor = \lceil -x \rceil \tag{3.3}$$

$$-\lceil x \rceil = \lfloor -x \rfloor \tag{3.4}$$

Para qualquer nmero real $x\ge 0$ e inteiros $a,b\gt 0$, temos

$$\lceil \dfrac{\lceil x/a \rceil}{b}\rceil = \lceil \dfrac{x}{ab}\rceil \tag{3.5}$$

$$\lfloor \dfrac{\lfloor x/a \rfloor}{b}\rfloor = \lfloor \dfrac{x}{ab}\rfloor \tag{3.6}$$

$$\lceil \dfrac{a}{b} \rceil \le \dfrac{a+(b-1)}{b} \tag{3.7}$$

$$\lfloor \dfrac{a}{b} \rfloor \ge \dfrac{a-(b-1)}{b} \tag{3.7}$$

Para qualquer inteiro $n$ e nmero real $x$, temos

$$\lfloor n+x \rfloor = n + \lfloor x \rfloor \tag{3.8}$$

$$\lceil n+x \rceil = n + \lceil x \rceil \tag{3.9}$$

Aritmtica modular

Para qualquer inteiro $a$ e qualquer inteiro positivo $n$, o valor de $a\bmod n$ o resto do quociente $a/n$:

$$a\bmod n = a - n\lfloor a / n\rfloor\tag{3.11}$$

$$0\le a\bmod n \lt n\tag{3.12}$$

Se $(a\bmod n) = (b\bmod n)$, escrevemos $a = b\pmod n$, ou seja, $a$ e $b$ tm o mesmo resto quando divididos por $n$. Equivalentemente, $n$ um divisor de $b-a$.

Polinomiais

Dizemos que uma funo $f(n)$ limitada por um polinmio se $f(n) = O(n^k)$ para alguma constante $k$.

Exponenciais

Para todos os reais $a > 0$, $m$ e $n$, temos as seguintes identidades:

  • $a^0=1$
  • $a^1=a$
  • $a^{-1}=1/a$
  • $(a^m)^n=a^{mn}$
  • $(a^m)^n=(a^n)^m$
  • $a^ma^n=a^{m+n}$
  • $a^m/a^n=a^{m-n}$
  • $(ab)^n=a^nb^n$
  • $(a/b)^n=a^n/b^n$
  • $(a/b)^{-n}=(b/a)^n$
  • $a^{m/n}=\sqrt[n]{a^m}$

Para todas as constantes reais $a > 1$ e $b$, temos:

$$n^b=o(a^n)\tag{3.13}$$

Seja $e$ o nmero de Euler. Para todos os reais $x$, temos:

$$e^x=1+x+\dfrac{x^2}{2!}+\dfrac{x^3}{3!}+\dots=\sum_{i=0}^\infin\dfrac{x^i}{i!}$$

$$1+x\le e^x\tag{3.14}$$

$$1+x\le e^x\le 1+x+x^2\tag{3.15}$$

$$\lim_{n\to\infin}(1+\dfrac{x}{n})^n=e^x\tag{3.16}$$

Logaritmos

Para todos os reais $a > 0$, $b > 0$, $c > 0$ e $n$, temos:

$$a=b^{\log_ba}\tag{3.17}$$

$$\log_c(ab)=\log_ca+\log_cb\tag{3.18}$$

$$\log_ba^n=n\log_ba$$

$$\log_ba=\dfrac{\log_ca}{\log_cb}\tag{3.19}$$

$$\log_b(1/a)=-\log_ba\tag{3.20}$$

$$\log_ba=\dfrac{1}{\log_ab}$$

$$a^{\log_bc}=c^{\log_ba}\tag{3.21}$$

Quando $|x|\lt 1$, temos:

$$\ln(1+x)=x-\dfrac{x^2}{2}+\dfrac{x^3}{3}-\dfrac{x^4}{4}+\dfrac{x^5}{5}-\dots \tag{3.22}$$

Para $x>-1$, temos:

$$\dfrac{x}{1+x}\le \ln(1+x)\le x \tag{3.23}$$

Dizemos que uma funo $f(n)$ limitada por um polilogaritmo se $f(n) = O(\lg^k n)$ para alguma constante $k$.

Para todas as constantes reais $a > 0$ e $b$, temos:

$$\lg^bn=(\lg n)^b=o(n^a)\tag{3.24}$$

Dessa forma, qualquer funo polinomial positiva cresce mais rpido do que qualquer funo polilogartmica.

Fatoriais

A aproximao de Stirling dada pela frmula

$$n! = \sqrt{2\pi n} \left(\frac{n}{e}\right)^n \left(1+\Theta\left(\frac{1}{n}\right)\right) \tag{3.25}$$

$$n!=o(n^n) \tag{3.26}$$

$$n!=\omega(2^n) \tag{3.27}$$

$$\lg(n!)=\Theta(n\lg n) \tag{3.28}$$

Para todos $n\ge1$, temos

$$n! = \sqrt{2\pi n} \left(\frac{n}{e}\right)^n e^{\alpha_n} \tag{3.29}$$

onde

$$\dfrac{1}{12n+1}\lt\alpha_n\lt\dfrac{1}{12n}$$

Iterao funcional

Usamos a notao $f^{(i)}(n)$ para denotar a funo $f(n)$ aplicada iterativamente $i$ vezes a um valor inicial de $n$, ou seja,

$$ \begin{aligned} f^{(i)}(n) &= \begin{cases} n & \text{se } i = 0 \\ f(f^{(i-1)}(n)) & \text{se } i \gt 0 \end{cases} \\ \end{aligned} $$

Por exemplo, se $f(n) = 2n$, ento $f^{(i)}(n) = 2^in$.

Nmeros de Fibonacci

$$ \begin{aligned} F_i &= \begin{cases} 0 & \text{se } i = 0 \\ 1 & \text{se } i = 1 \\ F_{i-1}+F_{i-2} & \text{se } i \ge 2 \end{cases} \\ \end{aligned} $$

Nmeros de Fibonacci esto relacionados proporo urea $\phi$ e seu conjugado $\widehat{\phi}$, que so as duas razes da equao $x^2 = x + 1$.

$$\phi=\dfrac{1+\sqrt 5}{2}=1.61803\cdots \tag{3.32}$$

$$\widehat{\phi}=\dfrac{1-\sqrt 5}{2}=-.61803\cdots \tag{3.33}$$

$$F_i= \dfrac{\phi^i-\widehat{\phi}^i}{\sqrt 5} = \left\lfloor \dfrac{\phi^i}{\sqrt 5}+\dfrac 1 2 \right\rfloor \tag{3.34}$$

Ou seja, o $i$-simo nmero de Fibonacci $F_i$ igual a $\phi/\sqrt 5$ arredondado para o inteiro mais prximo. Assim, os nmeros de Fibonacci crescem exponencialmente.

  9. Mostre que $k\lg k = \Theta(n)$ implica $k = \Theta(n / \lg n)$.

    10. Pela simetria de $\Theta$,

    $$k\lg k = \Theta(n) \Rightarrow n = \Theta(k\lg k)$$

    Uma vez que $n = \Theta(k\lg k)$, existem constantes positivas $c_1$, $c_2$ e $n_0$ tais que, para todo $n \geq n_0$, temos

    $$ \begin{gathered} c_1k\lg k \le n \le c_2 k \lg k \\ \lg(c_1k\lg k) \le \lg n \le \lg(c_2 k \lg k) && \text{(aplicado lg)} \\ \lg c_1 + \lg k + \lg \lg k \le \lg n \le \lg c_2 + \lg k + \lg \lg k && \text{(aplicada a equao (3.18))} \\ c_3 \lg k \le \lg n \le c_4 \lg k && \text{(desconsiderados menores termos)}\\ \lg n = \Theta(\lg k) \end{gathered} $$

    Uma vez que $k \lg k = \Theta(n)$, existem constantes positivas $c_1$, $c_2$ e $n_0$ tais que, para todo $n \geq n_0$, temos

    $$ \begin{aligned} c_1n &\leq k\lg k &\leq c_2n \\ \dfrac{c_1n}{\lg k} &\leq k &\leq \dfrac{c_2n}{\lg k} && \text{(dividido por $\lg k$)} \end{aligned} $$

    Uma vez que $\lg n = \Theta(\lg k)$, existem constantes positivas $d_1$, $d_2$ e $n_1$ tais que, para todo $n \geq n_1$, temos

    $d_1\lg k \leq \lg n \leq d_2\lg k$

    Isolando $\lg k$, temos $\lg k \le \dfrac{\lg n}{d_1}$ e $\lg k \ge \dfrac{\lg n}{d_2}$

    Uma vez que um denominador maior resulta em uma frao menor, e vice-versa, temos

    $\dfrac{c_1n}{\lg n / d_2} \le \dfrac{c_1n}{\lg k} \leq k \leq \dfrac{c_2n}{\lg k} \le \dfrac{c_2n}{\lg n / d_1}$

    Substituindo essas limitaes na inequao anterior, obtemos

    $\dfrac{c_1d_2n}{\lg n} \leq k \leq \dfrac{c_2d_1n}{\lg n}$

    O que implica que $k = \Theta(n / \lg n)$.
