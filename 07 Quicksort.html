Quicksort

7.1 Descrição do quicksort

Estratégia "dividir para conquistar":

Divide o array em dois lados, escolhendo um elemento pivô, e colocando todos os elementos menores que o pivô à sua esquerda (low side), e os maiores à sua direita (high side).

Conquiste recursivamente realizando o passo anterior.

Combine não fazendo nada: Essa etapa não é necessária...

A seguir, o algoritmo quicksort:

QUICKSORT(A, p, r)
1 if p < r
2     // Partition the subarray around the pivot, which ends up in A[q].
3     q = PARTITION(A, p, r)
4     QUICKSORT(A, p, q - 1) // recursively sort the low side
5     QUICKSORT(A, q + 1, r) // recursively sort the high side

Como alternativa ingênua para escolha do pivô, comecemos com o último elemento do array. A ideia é manter o (sub)array em três partes, a parte baixa, depois, a parte alta, e em seguida, a parte não verificada, que inicialmente começa sendo o (sub)array inteiro. Percorremos a parte não verificada fazendo a seguinte verificação: se o próximo elemento não verificado for baixo, troque-o com o primeiro elemento alto, aumentando, assim, a parte baixa. A seguir, o algoritmo de partição:

PARTITION(A, p, r)
1 x = A[r]                        // the pivot
2 i = p - 1                       // highest index into the low side
3 for j = p to r - 1              // process each element other than the pivot
4     if A[j] ≤ x                 // does this element belong on the low side?
5         i = i + 1               // index of a new slot in the low side
6         exchange A[i] with A[j] // put this element there
7 exchange A[i + 1] with A[r]     // pivot goes just to the right of the low side
8 return i + 1                    // new index of the pivot

Antes de cada iteração, para qualquer índice $k$, as seguintes propriedades são verdadeiras:

  1. Se $p\le k \le i$, então $A[k]\le x$ (parte baixa contém somente elementos menores que o pivô);
  2. Se $i+1\le k \le j-1$, então $A[k]\gt x$ (parte alta contém somente elementos maiores que o pivô);
  3. Se $k=r$, então $A[k]=x$ (pivô).

Provamos que essa invariante de laço é verdadeira antes da primeira iteração, que cada iteração a mantém verdadeira, e que o laço termina com a invariante sendo verdadeira:

Inicialização: Como $i\lt p$ e $i+1\lt j-1$, as duas primeiras propriedades são verdadeiras. A primeira linha do algoritmo satisfaz a terceira propriedade.

Manutenção: Se $A[j]\gt x$ (Figura 7.3a), então $A[j]$ é colocado na parte alta simplesmente incrementado $j$. Como $i$ não foi alterado, a primeira propriedade se manteve, e a segunda propriedade se tornou válida. Agora, se $A[j]\le x$ (Figura 7.3b), $i$ é incrementado para aumentar a parte baixa, e $A[i]$ é trocado com $A[j]$, para colocar o atual elemento no final da parte baixa, depois, incremente $j$ para restabelecer a segunda propriedade.

Término: O laço termina pois inicialmente $j\lt r$, e a cada iteração $j$ é incrementado até chegar a $r$.

7.2 Desempenho do quicksort

Quão mais no centro da partição ficar o pivô melhor o desempenho do quicksort. Quanto mais nas extremidades ficar o pivô, pior o desempenho do quicksort.

Pior caso

No pior caso, o pivô é o maior ou o menor elemento do array. Neste caso, uma partição terá todos os elementos exceto o pivô e a outra partição não terá nenhum elemento. Conforme o exercício 7.1-3, o custo de particionar é $\Theta(n)$. Assim, a recorrência do quicksort é:

$$T(n)=T(n-1)+T(0)+\Theta(n)=T(n-1)+\Theta(n)$$

Aplicando o método da substituição chegamos na solução $T(n)=\Theta(n^2)$.

O pior caso ocorre quando o array já está ordenado, curiosamente o melhor caso do insertion sort, que custa $O(n^2)$.

Melhor caso

No melhor caso, o pivô é o elemento mediano do array, isto é, o tamanho das partições é, respectivamente, $\lfloor(n-1)/2\rfloor$ e $\lceil(n-1)/2\rceil$. Como esses tamanho são menores que $n/2$, podemos escrever a recorrência simplificada:

$$T(n)=2T(n/2)+\Theta(n)$$

Aplicando o método da substituição chegamos na solução $T(n)=\Theta(n\lg n)$.

Caso médio

O custo do caso médio, como veremos, é mais próximo do melhor caso do que do pior caso. Para ter uma ideia, suponha que na média o particionamento produz sempre uma proporção de 9 para 1 entre as partições, o que parece ser bem ruim. Obtemos a recorrência:

$$T(n)=T(9n/10)+T(n/10)+\Theta(n)$$

A Figura 7.4 ilustra a árvore de recursão dessa recorrência. Cada nível custa $n$. A folha menos profunda ocorre no nível $\log_10 n=\Theta(\lg n)$. A partir desse nível, o custo de cada nível é no máximo $n$. A recursão termina no nível $\log_{10/9} n=\Theta(\lg n)$. Portanto apesar de não ser intuitivo, a soma de todos os níveis ainda custa $O(n\lg n)$. De fato, mesmo numa divisão de 99 para 1, ou qualquer divisão constante, a árvore de recursão terá uma profundidade de $\Theta(\lg n)$, e o custo da recursão será $O(n\lg n)$, apesar que a constante escondida pela notação assintótica será bem grande.

Intuição para o caso médio

Ao rodar o quicksort num array aleatório, esperamos que alguma partições sejam razoavelmente balanceadas enquanto que outras sejam desbalanceadas.

Suponha, para efeitos de comparação, que o melhor caso e o pior caso são alternados nas partições de cada nível. A Figura 7.5a ilustra uma partição ruim seguida por uma boa e a Figura 7.5b ilustra uma partição boa. Na Figura 7.5a, o custo de partição combinado de dois níveis (nós azuis) é $\Theta(n)+\Theta(n-1)=\Theta(n)$. Na Figura 7.5b, o custo de partição de um nível é $\Theta(n)$. Ainda os subproblemas a serem particionados da Figura 7.5a não são maiores que os subproblemas da Figura 7.5b. Ou seja, a Figura 7.5a é no máximo um fator constante mais lento que a Figura 7.5b. Portanto, o tempo de execução do quicksort alternando entre partições boas e ruins ainda é $O(n\lg n)$, com um fator constante pouco maior escondido pela notação assintótica se comparado a um quicksort que produz apenas as melhores partições.

  5. Suponha que as divisões em cada nível do quicksort estejam na proporção constante $\alpha$ para $\beta$, onde $\alpha+\beta=1$ e $0\lt \alpha \le \beta \lt 1$. Mostre que a profundidade mínima de uma folha na árvore de recursão é aproximadamente $\lg_{1/\alpha}n$ e que a profundidade máxima é aproximadamente $\lg_{1/\beta}n$. (Não se preocupe com o arredondamento inteiro.)

    6. A profundidade mínima segue um caminho que sempre escolhe a parte menor da partição, ou seja, que multiplica o número de elementos por $\alpha$. Cada nível de recursão reduz o número de elementos de $n$ para $\alpha n$, e $i$ níveis de recursão reduzem o número de elementos para $\alpha^i n$. Em uma folha, resta apenas um elemento, então em uma folha de profundidade mínima de $m$, temos $\alpha^m n = 1$. Dividindo ambos os lados por $n$ e aplicando logaritmo, temos:

    $$\begin{aligned} m&=\log_\alpha 1/n &\\ &=-\log_\alpha n &\text{propriedade (3.20)}\\ &=-\dfrac{\log_{1/\alpha} n}{\log_{1/\alpha} \alpha} &\text{propriedade (3.19)}\\ &=-\dfrac{\log_{1/\alpha} n}{-1} &(1/\alpha)^{-1}=\alpha\\ &=\log_{1/\alpha} n \end{aligned}$$

    Da mesma forma, o caminho de profundidade máxima corresponde a sempre escolher a parte maior da partição, ou seja, mantendo uma fração $\beta$ dos elementos a cada vez. A profundidade máxima $M$ é alcançada quando resta apenas um elemento, ou seja, quando $\beta^M n = 1$. Assim, $M = \log_{1/\beta}n$.

    Todas essas equações são aproximadas porque estamos ignorando pisos e tetos.

  6. Considere um array com elementos distintos e para o qual todas as permutações dos elementos são igualmente prováveis. Argumente que, para qualquer constante $0 \lt \alpha \le 1 / 2$, a probabilidade é aproximadamente $1 - 2\alpha$ de que $\text{PARTITION}$ produza uma divisão pelo menos tão balanceada quanto $1 - \alpha$ para $\alpha$.

    7. Para produzir uma divisão pior que $1 - \alpha$ para $\alpha$, $\text{PARTITION}$ deve escolher um pivô que esteja dentro dos $\alpha n$ elementos menores ou dos $\alpha n$ elementos maiores. A probabilidade de cada um é (aproximadamente) $\alpha n / n = \alpha$, e a probabilidade de ambos é $2\alpha$. Assim, a probabilidade de ter uma partição melhor é o complemento, $1 - 2\alpha$.
