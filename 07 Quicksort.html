Quicksort

7.1 Descrio do quicksort

Estratgia "dividir para conquistar":

Divide o array em dois lados, escolhendo um elemento piv, e colocando todos os elementos menores que o piv sua esquerda (low side), e os maiores sua direita (high side).

Conquiste recursivamente realizando o passo anterior.

Combine no fazendo nada: Essa etapa no necessria...

A seguir, o algoritmo quicksort:

QUICKSORT(A, p, r)
1 if p < r
2     // Partition the subarray around the pivot, which ends up in A[q].
3     q = PARTITION(A, p, r)
4     QUICKSORT(A, p, q - 1) // recursively sort the low side
5     QUICKSORT(A, q + 1, r) // recursively sort the high side

Como alternativa ingnua para escolha do piv, comecemos com o ltimo elemento do array. A ideia manter o (sub)array em trs partes, a parte baixa, depois, a parte alta, e em seguida, a parte no verificada, que inicialmente comea sendo o (sub)array inteiro. Percorremos a parte no verificada fazendo a seguinte verificao: se o prximo elemento no verificado for baixo, troque-o com o primeiro elemento alto, aumentando, assim, a parte baixa. A seguir, o algoritmo de partio:

PARTITION(A, p, r)
1 x = A[r]                        // the pivot
2 i = p - 1                       // highest index into the low side
3 for j = p to r - 1              // process each element other than the pivot
4     if A[j]  x                 // does this element belong on the low side?
5         i = i + 1               // index of a new slot in the low side
6         exchange A[i] with A[j] // put this element there
7 exchange A[i + 1] with A[r]     // pivot goes just to the right of the low side
8 return i + 1                    // new index of the pivot

Antes de cada iterao, para qualquer ndice $k$, as seguintes propriedades so verdadeiras:

  1. Se $p\le k \le i$, ento $A[k]\le x$ (parte baixa contm somente elementos menores que o piv);
  2. Se $i+1\le k \le j-1$, ento $A[k]\gt x$ (parte alta contm somente elementos maiores que o piv);
  3. Se $k=r$, ento $A[k]=x$ (piv).

Provamos que essa invariante de lao verdadeira antes da primeira iterao, que cada iterao a mantm verdadeira, e que o lao termina com a invariante sendo verdadeira:

Inicializao: Como $i\lt p$ e $i+1\lt j-1$, as duas primeiras propriedades so verdadeiras. A primeira linha do algoritmo satisfaz a terceira propriedade.

Manuteno: Se $A[j]\gt x$ (Figura 7.3a), ento $A[j]$ colocado na parte alta simplesmente incrementado $j$. Como $i$ no foi alterado, a primeira propriedade se manteve, e a segunda propriedade se tornou vlida. Agora, se $A[j]\le x$ (Figura 7.3b), $i$ incrementado para aumentar a parte baixa, e $A[i]$ trocado com $A[j]$, para colocar o atual elemento no final da parte baixa, depois, incremente $j$ para restabelecer a segunda propriedade.

Trmino: O lao termina pois inicialmente $j\lt r$, e a cada iterao $j$ incrementado at chegar a $r$.

7.2 Desempenho do quicksort

Quo mais no centro da partio ficar o piv melhor o desempenho do quicksort. Quanto mais nas extremidades ficar o piv, pior o desempenho do quicksort.

Pior caso

No pior caso, o piv o maior ou o menor elemento do array. Neste caso, uma partio ter todos os elementos exceto o piv e a outra partio no ter nenhum elemento. Conforme o exerccio 7.1-3, o custo de particionar $\Theta(n)$. Assim, a recorrncia do quicksort :

$$T(n)=T(n-1)+T(0)+\Theta(n)=T(n-1)+\Theta(n)$$

Aplicando o mtodo da substituio chegamos na soluo $T(n)=\Theta(n^2)$.

O pior caso ocorre quando o array j est ordenado, curiosamente o melhor caso do insertion sort, que custa $O(n^2)$.

Melhor caso

No melhor caso, o piv o elemento mediano do array, isto , o tamanho das parties , respectivamente, $\lfloor(n-1)/2\rfloor$ e $\lceil(n-1)/2\rceil$. Como esses tamanho so menores que $n/2$, podemos escrever a recorrncia simplificada:

$$T(n)=2T(n/2)+\Theta(n)$$

Aplicando o mtodo da substituio chegamos na soluo $T(n)=\Theta(n\lg n)$.

Caso mdio

O custo do caso mdio, como veremos, mais prximo do melhor caso do que do pior caso. Para ter uma ideia, suponha que na mdia o particionamento produz sempre uma proporo de 9 para 1 entre as parties, o que parece ser bem ruim. Obtemos a recorrncia:

$$T(n)=T(9n/10)+T(n/10)+\Theta(n)$$

A Figura 7.4 ilustra a rvore de recurso dessa recorrncia. Cada nvel custa $n$. A folha menos profunda ocorre no nvel $\log_10 n=\Theta(\lg n)$. A partir desse nvel, o custo de cada nvel no mximo $n$. A recurso termina no nvel $\log_{10/9} n=\Theta(\lg n)$. Portanto apesar de no ser intuitivo, a soma de todos os nveis ainda custa $O(n\lg n)$. De fato, mesmo numa diviso de 99 para 1, ou qualquer diviso constante, a rvore de recurso ter uma profundidade de $\Theta(\lg n)$, e o custo da recurso ser $O(n\lg n)$, apesar que a constante escondida pela notao assinttica ser bem grande.

Intuio para o caso mdio

Ao rodar o quicksort num array aleatrio, esperamos que alguma parties sejam razoavelmente balanceadas enquanto que outras sejam desbalanceadas.

Suponha, para efeitos de comparao, que o melhor caso e o pior caso so alternados nas parties de cada nvel. A Figura 7.5a ilustra uma partio ruim seguida por uma boa e a Figura 7.5b ilustra uma partio boa. Na Figura 7.5a, o custo de partio combinado de dois nveis (ns azuis) $\Theta(n)+\Theta(n-1)=\Theta(n)$. Na Figura 7.5b, o custo de partio de um nvel $\Theta(n)$. Ainda os subproblemas a serem particionados da Figura 7.5a no so maiores que os subproblemas da Figura 7.5b. Ou seja, a Figura 7.5a no mximo um fator constante mais lento que a Figura 7.5b. Portanto, o tempo de execuo do quicksort alternando entre parties boas e ruins ainda $O(n\lg n)$, com um fator constante pouco maior escondido pela notao assinttica se comparado a um quicksort que produz apenas as melhores parties.

  5. Suponha que as divises em cada nvel do quicksort estejam na proporo constante $\alpha$ para $\beta$, onde $\alpha+\beta=1$ e $0\lt \alpha \le \beta \lt 1$. Mostre que a profundidade mnima de uma folha na rvore de recurso aproximadamente $\lg_{1/\alpha}n$ e que a profundidade mxima aproximadamente $\lg_{1/\beta}n$. (No se preocupe com o arredondamento inteiro.)

    6. A profundidade mnima segue um caminho que sempre escolhe a parte menor da partio, ou seja, que multiplica o nmero de elementos por $\alpha$. Cada nvel de recurso reduz o nmero de elementos de $n$ para $\alpha n$, e $i$ nveis de recurso reduzem o nmero de elementos para $\alpha^i n$. Em uma folha, resta apenas um elemento, ento em uma folha de profundidade mnima de $m$, temos $\alpha^m n = 1$. Dividindo ambos os lados por $n$ e aplicando logaritmo, temos:

    $$\begin{aligned} m&=\log_\alpha 1/n &\\ &=-\log_\alpha n &\text{propriedade (3.20)}\\ &=-\dfrac{\log_{1/\alpha} n}{\log_{1/\alpha} \alpha} &\text{propriedade (3.19)}\\ &=-\dfrac{\log_{1/\alpha} n}{-1} &(1/\alpha)^{-1}=\alpha\\ &=\log_{1/\alpha} n \end{aligned}$$

    Da mesma forma, o caminho de profundidade mxima corresponde a sempre escolher a parte maior da partio, ou seja, mantendo uma frao $\beta$ dos elementos a cada vez. A profundidade mxima $M$ alcanada quando resta apenas um elemento, ou seja, quando $\beta^M n = 1$. Assim, $M = \log_{1/\beta}n$.

    Todas essas equaes so aproximadas porque estamos ignorando pisos e tetos.

  6. Considere um array com elementos distintos e para o qual todas as permutaes dos elementos so igualmente provveis. Argumente que, para qualquer constante $0 \lt \alpha \le 1 / 2$, a probabilidade aproximadamente $1 - 2\alpha$ de que $\text{PARTITION}$ produza uma diviso pelo menos to balanceada quanto $1 - \alpha$ para $\alpha$.

    7. Para produzir uma diviso pior que $1 - \alpha$ para $\alpha$, $\text{PARTITION}$ deve escolher um piv que esteja dentro dos $\alpha n$ elementos menores ou dos $\alpha n$ elementos maiores. A probabilidade de cada um (aproximadamente) $\alpha n / n = \alpha$, e a probabilidade de ambos $2\alpha$. Assim, a probabilidade de ter uma partio melhor o complemento, $1 - 2\alpha$.
