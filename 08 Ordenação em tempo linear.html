8 Ordenao em tempo linear

At o momento, vimos algoritmos que no pior caso custam $O(n\lg n)$. Ser que podemos implementar um algoritmo que no pior caso custe menos que isso?

Os algoritmos de ordenao vistos compartilham uma propriedade interessante: a ordenao feita com base somente na comparao entre os elementos de entrada. Chamamos tais algoritmos de ordenao por comparao.

8.1 Limites inferiores para ordenao

8.2 Counting sort

Counting sort assume que cada um dos $n$ elementos seja um inteiro entre $0$ e $k$, para algum inteiro $k$. Ele conta a quantidade de elementos menor ou igual a cada elemento, que nos d a posio desse elemento. Quando h elementos repetidos, devemos fazer ajustes para no ca-los todos na mesma posio.

A Figura 8.2 ilustra o counting sort. A descrio do algoritmo :

  1. Inicializa-se um vetor $C[0:k]$ com zero.
  2. Para cada elemento no vetor $A$, seu valor representa o ndice no vetor $C$. Esse valor ento incrementado, resultando em $C$ contendo a contagem de ocorrncias para cada elemento nico em $A$.
  3. Do primeiro ao ltimo elemento de $C$, acumula a contagem de elementos menores que o elemento atual, que so os da sua esquerda.
  4. Do ltimo ao primeiro elemento de $A$, sua posio final ser seu valor correspondente em $C$. Para lidar com duplicatas, o valor em $C$ decrementado aps cada alocao.

O pseudocdigo do algoritmo :

COUNTING-SORT(A, n, k)
 1 let B[1 : n] and C [0 : k] be new arrays
 2 for i = 0 to k
 3     C [i] = 0
 4 for j = 1 to n
 5     C [A[j]] = C [A[j]] + 1
 6 // C [i] now contains the number of elements equal to i.
 7 for i = 1 to k
 8     C [i] = C [i] + C [i - 1]
 9 // C [i] now contains the number of elements less than or equal to i.
10 // Copy A to B, starting from the end of A.
11 for j = n downto 1
12     B[C [A[j]]] = A[j]
13     C [A[j]] = C [A[j]] - 1 // to handle duplicate values
14 return B

Seu tempo de execuo $\Theta(n+k)$. Na prtica, geralmente usamos counting sort quando $k=O(n)$, pois assim seu tempo de execuo $\Theta(n)$.

Counting sort consegue vencer o limite inferior $O(n\lg n)$ de pior caso de ordenao, pois ele no compara os elementos entre si. Na verdade, ele usa os elementos como ndices para outro array.

Uma propriedade importante do counting sort ser estvel: Se antes da ordenao, um elemento $x$ vinha antes de um elemento $y$ com mesma chave, aps a ordenao, $x$ continua antes de $y$.

Exerccios

  1. Utilizando a Figura 8.2 como modelo, ilustre a operao do $\text{COUNTING-SORT}$ no array $A = \langle 6, 0, 2, 0, 1, 3, 4, 6, 1, 3, 2 \rangle$.
  2. Prove que $\text{COUNTING-SORT}$ estvel.
  3. Suponha que estivssemos reescrevendo o cabealho do lao for na linha 11 do $\text{COUNTING-SORT}$ como 
    11  para j = 1 at n

    Mostre que o algoritmo ainda funciona corretamente, mas que no estvel. Em seguida, reescreva o pseudocdigo para o counting sort de modo que os elementos com o mesmo valor sejam escritos no array de sada na ordem dos ndices crescentes, tornando o algoritmo estvel.

    4. O algoritmo correto, quer $A$ seja processado da frente para trs ou de trs para frente.

    No entanto, o algoritmo modificado no estvel. Como antes, no ltimo lao for, um elemento igual ao retirado de $A$ anteriormente colocado antes do anterior (ou seja, em uma posio de ndice inferior) no array de sada $B$. O algoritmo original era estvel porque um elemento retirado de $A$ posteriormente comeava com um ndice inferior ao de um retirado anteriormente. Mas no algoritmo modificado, um elemento retirado de $A$ posteriormente comea com um ndice superior ao de um retirado anteriormente.

    Em particular, o algoritmo ainda coloca os elementos com valor $k$ nas posies $C[k-1] + 1$ at $C[k]$, mas na ordem reversa de sua ocorrncia em $A$.

    Poderamos pensar em estender esse algoritmo colocando os elementos de $A$ em uma coleo de elementos para cada clula no array $C$. Ento, se usarmos uma coleo FIFO, a modificao na linha 10 tornar o algoritmo estvel; se usarmos uma coleo LILO, ele ser antiestvel.

  4. Prove a seguinte invariante de lao para o $\text{COUNTING-SORT}$:

    No incio de cada iterao do lao for nas linhas 1113, o ltimo elemento em $A$ com valor $i$ que ainda no foi copiado para $B$ deve estar em $B[C[i]]$.

  5. Suponha que o array sendo ordenado contenha apenas inteiros no intervalo de $0$ a $k$, e que no h dados satlites para serem movidos com essas chaves. Modifique o counting sort para usar apenas os arrays $A$ e $C$, colocando o resultado ordenado de volta no array $A$ em vez de em um novo array $B$.
  6. Descreva um algoritmo que, dado $n$ inteiros no intervalo de $0$ a $k$, realize um pr-processamento em sua entrada e depois responda qualquer consulta sobre quantos dos $n$ inteiros esto dentro de um intervalo $[a..b]$ em tempo $O(1)$. Seu algoritmo deve usar tempo de pr-processamento $\Theta(n + k)$.
  7. Counting sort tambm pode funcionar eficientemente se os valores de entrada tiverem partes fracionrias, mas o nmero de dgitos na parte fracionria for pequeno. Suponha que voc tenha $n$ nmeros no intervalo de $0$ a $k$, cada um com no mximo $d$ dgitos decimais (base 10) direita do ponto decimal. Modifique o counting sort para rodar em tempo $\Theta(n + 10^d k)$.
