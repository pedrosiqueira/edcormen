Estruturas de Dados

Introduo

Enquanto os conjuntos matemticos so invariveis, os conjuntos manipulados por algoritmos podem crescer, encolher ou sofrer outras mudanas ao longo do tempo. Chamamos tais conjuntos de conjuntos dinmicos.

Damos o nome de dicionrio ao conjunto dinmico em que cada elemento, chamado de chave, possui um valor associado. Esse nome vem do termo dicionrio, em que procuramos pelas definies (valor) das palavras (chave). Um outro exemplo de uma estrutura do tipo dicionrio seria o ndice de um livro, que lista as palavras-chaves (chave), com as pginas em que ocorrem (valor).

O elemento pode conter dados satlites, que so outros atributos alm da chave que no so utilizados pela implementao do conjunto.

Apresentamos a seguir, uma lista de operaes tpicas em conjuntos dinmicos:

  • search(S, k) - Uma consulta que, dado um conjunto S e uma chave k, retorna um ponteiro x para um elemento em S tal que x.key = k ou NIL se nenhum elemento desse tipo pertencer a S.
  • insert(S, x) - Uma operao modificadora que aumenta o conjunto S com o elemento apontado por x.
  • delete(S, x) - Uma operao modificadora que, dado um ponteiro x para um elemento no conjunto S, exclui x de S. (Observe que essa operao utiliza um ponteiro para um elemento x, no um valor de chave.)
  • mininum(S) - Uma consulta em um conjunto ordenado S que retorna um ponteiro para o elemento de S que tenha a menor chave.
  • maximum(S) - Uma consulta em um conjunto ordenado S que retorna um ponteiro para o elemento de S que tenha a maior chave.
  • successor(S, x) - Uma consulta que, dado um elemento x cuja chave de um conjunto ordenado S, retorna um ponteiro para o menor elemento aps x em S ou NIL se x o elemento mximo.
  • predecessor(S, x) - Uma consulta que, dado um elemento x, cuja chave de um conjunto ordenado S, retorna um ponteiro para o maior elemento antes de x em S ou NIL se x o elemento mnimo.

Estruturas de dados elementares

10.1 Estruturas de dados baseadas em array

10.1.1 Arrays

Um array armazenado como uma sequncia contgua de bytes na memria. Devido a esse fato, possvel calcular em tempo constante a localizao de um elemento na memria sabendo o seu ndice. Isto , leva tempo constante acessar qualquer elemento pelo seu ndice.

Isso implica que os elementos do array devem ocupar o mesmo espao na memria. Quando no for o caso, o array armazena endereos de memria que apontam para objetos, que j no ficam armazenados diretamente no array. Ainda assim, o acesso a um objeto, sabendo seu ndice no array, continua constante.

10.1.2 Matrizes

Tipicamente representamos matrizes como arrays bidimensionais. Mas tambm podemos ter matrizes multidimensionais. Como a memria basicamente uma sequncia contgua de bytes, matrizes de fato so armazenadas como arrays, e ao acessar um elemento atravs dos ndices de linha e coluna, o computador faz uma aritmtica para saber o ndice na memria.

Pilhas e filas

Numa pilha, o elemento inserido vai para o topo da pilha, e s pode ser excludo o elemento que est no topo. Essa poltica de excluso chamada de LIFO, ou last in, first out. Numa fila, o elemento inserido vai para o fim da fila, e s pode ser excludo o elemento que est no incio da fila. J essa poltica FIFO, ou first in, first out.

Pilhas

Quando S.top=0, a pilha est vazia. No precisamos zerar as posies do array, consideramos o que est l como lixo. Tentar inserir um elemento numa pilha cheia ocorre stack overflow (estouro de pilha, ou transbordamento de pilha), e tentar excluir um elemento de uma pilha vazia ocorre stack underflow (escassez de pilha, ou esgotamento de pilha).

STACK-EMPTY(S)
1 if S.top == 0
2      return TRUE
3 else return FALSE

PUSH(S, x)
1 if S.top == S.size
2      error overflow
3 else S.top = S.top + 1
4      S[S.top] = x

POP(S)
1 if STACK-EMPTY(S)
2      error underflow
3 else S.top = S.top - 1
4      return S[S.top + 1]

Todas as operaes tm custo constante.

A fila segue uma ordem circular. Quando Q.head==Q.tail, a fila est vazia. Quando Q.head==Q.tail+1 ou Q.head==1 e Q.tail=Q.size, a fila est cheia. Na fila tambm pode ocorrer underflow ou overflow. O seguinte cdigo no evita underflow e overflow, que so deixados como exerccios.

ENQUEUE(Q, x)
1 Q[Q.tail] = x
2 if Q.tail == Q.size
3      Q.tail = 1
4 else Q.tail = Q.tail + 1

DEQUEUE(Q)
1 x = Q[Q.head]
2 if Q.head == Q.size
3      Q.head = 1
4 else Q.head = Q.head + 1
5 return x

Todas as operaes tm custo constante.

10.2 Listas encadeadas

Em listas encadeadas, ao invs dos elementos ocuparem posies contguas na memria como em arrays, os elementos so apontados uns pelos outros consecutivamente. Tambm so chamadas de listas ligadas.

A Figura 10.4 ilustra uma lista duplamente encadeada, em que cada n aponta tanto para o prximo n quanto para o n anterior.

Elementos so objetos que contm o campo key e os demais dados satlites. A forma de implementao da lista pode considerar o n sendo o elemento, ou, considerar o n contendo o elemento. Por simplificao, no livro, o n ser o elemento.

Na lista encadeada simples, cada n aponta apenas para o prximo n, sem apontar para o anterior. Na lista encadeada circular o n anterior a head o ltimo n da lista. A lista pode ser ordenada ou no, o que afeta a posio em que os ns podem ser inseridos.

Buscando em uma lista encadeada

Algoritmo:

LIST-SEARCH(L, k)
1 x = L.head
2 while x  NIL and x.key  k
3   x = x.next
4 return x

O custo $\Theta(n)$ no pior caso, quando o elemento desejado est no final da lista.

Inserindo em uma lista encadeada

Algoritmo de insero no incio:

LIST-PREPEND(L, x)
1 x.next = L.head
2 x.prev = NIL
3 if L.head  NIL
4   L.head.prev = x
5 L.head = x

O custo $\Theta(1)$.

Algoritmo de insero de um novo n x na frente de um n y:

LIST-INSERT(x, y)
1 x.next = y.next
2 x.prev = y
3 if y.next  NIL
4   y.next.prev = x
5 y.next = x

Como j temos o n y, o custo de insero $\Theta(1)$. Mas para buscarmos y pela chave, o custo de pior caso $\Theta(n)$.

Excluso em uma lista encadeada

Algoritmo de excluso de um n x:

LIST-DELETE(L, x)
1 if x.prev  NIL
2      x.prev.next = x.next
3 else L.head = x.next
4 if x.next  NIL
5      x.next.prev = x.prev

Como j temos o n x, o custo de excluso $\Theta(1)$. Mas para buscarmos x pela chave, o custo de pior caso $\Theta(n)$.

Sentinela

Uma sentinela um n que representa NIL. O primeiro e o ltimo n da lista, ao invs de apontar para NIL, aponta para a sentinela. Nessa configurao, no precisamos mais de uma referncia a uma lista com seu campo head, apenas sentinela. E a lista se torna duplamente encadeada circular. A Figura 10.5 ilustra a lista com sentinela:

Na Figura 10.5, a sentinela representada pelo n azul, e fica entre o head e o tail.

Algoritmo de excluso adaptado:

LIST-DELETE(x)
1 x.prev.next = x.next
2 x.next.prev = x.prev

Agora podemos ignorar os limites de head e tail da lista. S temos que cuidar para no excluir a sentinela, pois assim perderamos a referncia para a lista.

Algoritmo de insero adaptado:

LIST-INSERT(x, y)
1 x.next = y.next
2 x.prev = y
3 y.next.prev = x
4 y.next = x

Agora para inserir no incio (head), y a sentinela, e para inserir no fim (tail), y o n anterior sentinela.

Nessas adaptaes economizamos as intrues condicionais mas os algoritmos continuam com o mesmo custo assinttico $O(1)$.

Algoritmo de busca adaptado:

LIST-SEARCH(x, k)
1 x = x.next // start at the head of the list
2 while x.key  NIL and x.key  k
3      x = x.next
4 if x.key == NIL // did not find k
5      return NIL
6 else return x // found k in element x

Para diferenciar a sentinela dos demais ns, consideramos que sua chave NIL, isto , nenhum outro n pode ter essa mesma chave. Alm do mais, esse algoritmo considera que x inicialmente seja a sentinela, para garantir que todos os elementos sejam averiguados.

Nessa adaptao, acrescentamos algumas instrues mas o algoritmo continua com o mesmo custo assinttico $\Theta(n)$.
