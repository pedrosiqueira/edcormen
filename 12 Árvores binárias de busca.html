12 rvores binrias de busca

Estruturas de dados do tipo "rvores de busca" suportam as seguintes operaes de conjuntos dinmicos: insero, busca, excluso, menor, maior, predecessor e sucessor. Voc pode usar rvores de busca tanto como dicionrios quanto como filas de prioridades.

12.1 O que uma rvore binria de busca?

Implementamos uma rvore binria usando uma estrutura encadeada. Alm da chave e dados satlites, cada n contm os atributos left, right e p, que apontam, respectivamente, para seus filhos esquerdo, direito e seu pai. Se no houver um filho ou pai, o respectivo atributo aponta pra NIL. A prpria rvore possui o atributo root, que aponta para o n raiz, ou NIL, se a rvore estiver vazia. O n raiz o nico n da rvore cujo pai NIL.

Para qualquer n $x$, as chaves da subrvore esquerda de $x$ so menores ou iguais chave de $x$, e as chaves da subrvore direita de $x$ so maiores ou iguais chave de $x$. Ambas as rvores da Figura 12.1 possuem os mesmos objetos.

Devido essa propriedade da rvore binria de busca (BST, binary search tree), voc pode imprimir ordenadamente todas as chaves da rvore usando o percurso em-ordem:

INORDER-TREE-WALK(x)
1 if x  NIL
2   INORDER-TREE-WALK(x.left)
3   print x.key
4   INORDER-TREE-WALK(x.right)

A chamada inicial para imprimir ordenadamente todos os ns de uma rvore $T$ INORDER-TREE-WALK(T.root).

Similarmente, se imprimirmos a chave do n atual antes das chaves de seus filhos, temos o percurso pr-ordem, e se imprimirmos a chave do n atual depois de imprimir as chaves de seus filhos, temos o percurso ps-ordem.

Teorema 12.1

Se $x$ a raiz de uma subrvore com $n$ ns, ento INORDER-TREE-WALK(x) custa $\Theta(n)$.

Prova: Seja $T(n)$ o tempo do algoritmo para percorrer todos os $n$ ns da subrvore. Como o algoritmo visita todos os ns da subrvore, temos que $T(n)=\Omega(n)$. Basta provar que $T(n)=O(n)$.

Para uma subrvore vazia, o algoritmo leva um pequeno tempo constante, portanto, $T(0)=c$, para alguma constante $c\gt 0$.

Para $n\gt 0$, suponha que a subrvore esquerda de $x$ possua $k$ ns, tendo, portanto, a subrvore direita de $x$, $n-k-1$ ns. O tempo do algoritmo limitado por $T(n)\le T(k)+T(n-k-1)+d$, para alguma constante $d\gt 0$ que reflete o tempo de execuo do corpo do algoritmo.

Usando o mtodo da substituio, mostramos que $T(n)=O(n)$ provando que $T(n)=(c+d)n+c$. Para $n=0$, temos $(c+d)\cdot 0+c=c=T(0)$. Para $n\gt 0$, temos:

$$\begin{aligned} T(n) &\le T(k) + T(n-k-1)+d \\ &\le ((c+d)k+c)+((c+d)(n-k-1)+c)+d \\ &= (c+d)n+c-(c+d)+c+d \\ &=(c+d)n+c \end{aligned}$$

  3. D um algoritmo no recursivo que percorre a rvore em ordem. ($\textit{Dica:}$ Uma soluo fcil usa uma pilha como estrutura de dados auxiliar. Uma soluo mais complicada, mas elegante, no usa uma pilha, mas assume que podemos testar dois ponteiros quanto igualdade.)

  5. Argumente que, como ordenar $n$ elementos leva tempo $\Omega(n\lg n)$ no pior caso no modelo de comparao, qualquer algoritmo baseado em comparao para construir uma rvore binria de busca a partir de uma lista arbitrria de $n$ elementos leva tempo $\Omega(n\lg n)$ no pior caso.

    6. Esse exerccio pergunta se possvel inserir $n$ elementos em uma rvore inicialmente vazia em tempo de pior caso menor que $\Omega(n\lg n)$. A resposta no, pois, por contradio, se fosse possvel inserir em menos tempo, o percurso em-ordem custa $O(n)$, assim, obteramos as chaves ordenadas em tempo menor que $\Omega(n\lg n)$ no pior caso, que comprovadamente impossvel.

12.2 Consultando BSTs

12.2.1 Busca

Dado um ponteiro $x$ para um n da rvore, e uma chave $k$, a busca procura pelo n com chave $k$ na rvore de $x$ para baixo. Isto , se $x$ for $T.root$, a busca se dar na rvore inteira.

O procedimento traa um caminho da raiz at o n procurado, ou ento, at NIL, caso no encontre o n. Como os ns tm uma relativa ordem entre si, descendo pelo caminho, podemos deduzir se devemos ir pela direita ou pela esquerda. A Figura 12.2a ilustra a busca pelo n 13.

TREE-SEARCH(x, k)
1 if x == NIL or k == x.key
2   return x
3 if k < x.key
4   return TREE-SEARCH(x.left, k)
5 return TREE-SEARCH(x.right, k)

Os ns varridos durante a recurso formam um caminho simples descendo pela raiz da rvore e, portanto, o tempo de execuo $O(h)$, sendo $h$ a altura da rvore.

12.2.2 Menor e maior

Para encontrar o menor elemento, basta descer pela esquerda at chegar numa folha. Similarmente para o maior elemento, basta descer pela direita. A propriedade de BST garante que esses procedimentos so corretos. A Figura 12.2b ilustra esses procedimentos.

TREE-MINIMUM(x)
1 while x.left  NIL
2   x = x.left
3 return x

O procedimento para TREE-MAXIMUM simtrico. Ambos custam $O(h)$, j que descem pela rvore da raiz at uma de suas folhas.

12.2.3 Sucessor e predecessor

O sucessor de um n o que vem aps ele num percurso em-ordem. Mais formalmente, o sucessor de um n $x$ aquele que possui a menor chave maior ou igual chave do n $x$. O legal do procedimento que voc consegue encontrar o sucessor sem precisar comparar as chaves entre si.

O cdigo tem dois casos. Se a subrvore direita de $x$ no estiver vazia, ento o sucessor ser o menor elemento de sua subrvore direita, bastando chamar o procedimento TREE-MINIMUM(x.right). Por exemplo, o sucessor de $15$ na Figura 12.2c o $17$.

Agora, se $x$ no possuir uma subrvore direita, ento seu sucessor seu ancestral mais baixo cujo filho esquerdo tambm ancestral de $x$ (de outra forma, seu sucessor o primeiro ancestral, subindo pela rvore, que maior que ele). A Figura 12.2d ilustra esse caso para o n $13$.

TREE-SUCCESSOR(x)
1 if x.right  NIL               // case 1: leftmost node in right subtree
2   return TREE-MINIMUM(x.right)
3 y = x.p
4 while y  NIL and x == y.right // case 2: find the lowest ancestor of x whose left child is an ancestor of x
5   x = y
6   y = y.p
7 return y

O procedimento para TREE-PREDECESSOR simtrico. Ambos custam $O(h)$, j que, ou descem pela rvore at no mximo uma folha, ou sobem pela rvore at no mximo a raiz.

Teorema 12.2

As operaes de conjunto dinmico, TREE-SEARCH, TREE-MINIMUM, TREE-MAXIMUM, TREE-SUCCESSOR e TREE-PREDECESSOR levam tempo $O(h)$ em uma BST de altura $h$.

(Lembre-se de que todo n seu prprio ancestral.)
  7. Um mtodo alternativo para percorrer em ordem uma rvore binria de busca de $n$ ns encontrar o menor elemento na rvore chamando $\text{TREE-MINIMUM}$ e ento fazer $n - 1$ chamadas a $\text{TREE-SUCCESSOR}$. Prove que este algoritmo roda em tempo $\Theta(n)$.

    8. Observe que uma chamada a TREE-MINIMUMseguida por $n - 1$ chamadas a TREE-SUCCESSORrealiza exatamente o mesmo percurso em ordem que o procedimento INORDER-TREE-WALK. INORDER-TREE-WALKimprime primeiro o TREE-MINIMUM, e por definio, o TREE-SUCCESSORde um n o prximo n na ordem determinada pelo percurso em ordem da rvore.

    Esse algoritmo roda em tempo $\Theta(n)$ porque:

    • Requer tempo $\Omega(n)$ para realizar as $n$ chamadas de procedimento.
    • Percorre cada uma das $n - 1$ arestas da rvore no mximo duas vezes, o que leva tempo $O(n)$.

    Para ver que cada aresta percorrida no mximo duas vezes (uma vez descendo pela rvore e uma vez subindo), considere a aresta entre qualquer n $u$ e um de seus filhos, o n $v$. Ao comear pela raiz, o percurso deve descer de $u$ para $v$, antes de subir de $v$ para $u$.

    Suponha que $v$ o filho esquerdo de $u$.

    • Antes de imprimir $u$, o percurso deve imprimir todos os ns em sua subrvore esquerda, que est enraizada em $v$, garantindo a descida pela aresta $(u, v)$.
    • Depois que todos os ns na subrvore esquerda de $u$ so impressos, $u$ deve ser impresso em seguida. O procedimento TREE-SUCCESSORsobe at $u$ partindo do maior elemento (que no tem subrvore direita) na subrvore enraizada em $v$. Este caminho inclui claramente a aresta $(u, v)$, e como todos os ns na subrvore esquerda de $u$ foram impressos, a aresta $(u, v)$ no percorrida novamente.

    Agora suponha que $v$ seja o filho direito de $u$.

    • Depois que $u$ impresso, TREE-SUCCESSOR(u) chamado. Para chegar ao menor elemento na subrvore direita de $u$ (cuja raiz $v$), deve-se descer pela aresta $(u, v)$.
    • Depois que todos os valores na subrvore direita de $u$ so impressos, TREE-SUCCESSOR chamado no maior elemento (novamente, que no tem subrvore direita) na subrvore enraizada em $v$. TREE-SUCCESSORsobe pela rvore at um elemento aps $u$, j que $u$ j foi impresso. A aresta $(u, v)$ visitada nessa subida, e como todos os ns na subrvore direita de $u$ foram impressos, a aresta $(u, v)$ no percorrida novamente.

    Assim, nenhuma aresta percorrida duas vezes na mesma direo. Portanto, esse algoritmo roda em tempo $\Theta(n)$.

  8. Prove que, no importa em que n comeamos em uma rvore de busca binria de altura $h$, $k$ chamadas sucessivas para $\text{TREE-SUCCESSOR}$ leva tempo $O(k + h)$.
  9. Seja $T$ uma rvore de busca binria cujas chaves so distintas, seja $x$ um n folha e seja $y$ seu pai. Mostre que $y.key$ ou a menor chave em $T$ maior que $x.key$ ou a maior chave em $T$ menor que $x.key$.

12.3 Insero e excluso

12.3.1 Insero

A insero traa um caminho da raiz at um NIL, onde o novo n ser inserido, mantendo o ponteiro para o n anterior, que ser o pai do novo n.

Como o caminho da raiz at uma das folhas passa por $h$ ns, sendo $h$ a altura da rvore, o custo da insero $O(h)$.

TREE-INSERT(T, z)
1 x = T.root       // node being compared with z
2 y = NIL          // y will be parent of z
3 while x  NIL    // descend until reaching a leaf
4      y = x
5      if z.key < x.key
6           x = x.left
7      else x = x.right
8 z.p = y          // found the locationinsert z with parent y
9 if y == NIL	
10      T.root = z  // tree T was empty
11 else if z.key < y.key	
12      y.left = z
13 else y.right = z
